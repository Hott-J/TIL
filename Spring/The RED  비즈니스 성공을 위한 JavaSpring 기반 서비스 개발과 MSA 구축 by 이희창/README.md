### 프로젝트 구조 및 설계



#### 좋은 구현?

모든 개발과 구현의 목적은 유저의 경험을 만족시켜야하고, 회사가 목표로 하는 비지니스 가치를 모두 충족시켜야한다.

사용자는 신기술이네? 라고 하면서 쓰는게 아니라, 기능이 잘 동작하고, 배송이 잘되고, 잘 추천해주는 것을 중요시여김.

개발자로서 본인이 하고싶은것, 신기술이 과연 중요할까? 하드코딩을 하여 유저의 경험과 비지니스 목표를 빠르게 달성한다면 

괜찮다. (Trade-Off)

- 테스트 코드 작성 용이
- 가독성
